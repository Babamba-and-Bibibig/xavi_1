# User Report Template

이 템플릿은 사용자 문서 전담 AI 가 새 보고서성 문서를 만들 때 기본 뼈대로 사용한다.

## 가장 중요한 규칙

- 실제로 존재하는 코드와 문서에서 확인한 사실만 쓴다.
- 없는 기능을 추정해서 쓰지 않는다.
- 불확실하면 불확실하다고 적는다.
- 사람 독자가 빠르게 읽을 수 있게 핵심만 구조적으로 정리한다.

## 권장 구조

### 1. 요약

- 이번 문서가 설명하는 범위
- 사용자에게 중요한 핵심 변화 또는 핵심 구조

### 2. 현재 존재하는 기능

- 지금 실제로 구현되어 있는 기능만 적는다.
- 아직 구현되지 않은 항목은 따로 분리해 적거나 아예 적지 않는다.

### 3. 관련 파일

- 파일 경로
- 파일 목적
- 이 기능에서 맡는 책임

### 4. 동작 방식

- 이 기능이나 구조가 어떻게 동작하는지 단계적으로 설명
- 필요하면 입력, 처리, 출력 흐름으로 설명

### 5. 핵심 로직

- 중요한 규칙
- 상태 변화
- 분기 조건
- 사용자가 이해해야 할 포인트

### 6. 용어 설명

- 생소한 개념
- 프로젝트 안에서 특별한 의미로 쓰는 단어

### 7. 변경 영향

- 이번 변화로 사용자가 무엇을 이해해야 하는가
- 관련된 다른 파일이나 기능에 어떤 영향이 있는가

### 8. 아직 없는 것

- 현재 코드에서 확인되지 않는 기능
- 아직 구현되지 않은 부분
- 문서에 일부러 넣지 않은 추정 영역

## 금지

- TODO 나 내부 역할 메모를 섞어 쓰지 않는다.
- AI 세션용 체크리스트를 넣지 않는다.
- future plan 을 현재 구현처럼 쓰지 않는다.
- 코드에 없는 세부사항을 상상해서 채우지 않는다.
