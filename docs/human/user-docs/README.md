# User Docs System

이 폴더는 프로젝트 주제 주입 이후 사용자에게 보여줄 문서를 관리하는 공간이다.
현재 저장소는 특정 사용자 프로젝트가 아니라 AI 협업 부트스트랩이므로, 이 문서는 초기 상태 안내만 남긴다.

## 현재 상태

- 아직 특정 프로젝트 주제는 주입되지 않았다.
- 이 폴더의 문서는 프로젝트 개발이 시작된 뒤 사용자 관점 설명으로 채워진다.
- 개발이 시작된 뒤 `docs/human/user-docs/` 문서는 Git에 올라갈 수 있다.
- 이 문서 폴더를 `.gitignore` 로 숨기는 방식으로 초기 상태를 만들지 않는다.

## 문서 역할

- `README.md`: 사용자 문서 시스템의 기준과 현재 상태
- `feature-map.md`: 주제 주입 뒤 기능별 동작 방식
- `file-map.md`: 주제 주입 뒤 파일별 책임과 읽는 순서
- `live-log.md`: 주제 주입 뒤 사용자에게 의미 있는 변경 기록
- `report-template.md`: 사용자용 보고서 구조 템플릿

## 작성 기준

- 실제 코드, 문서, 검증 결과에 근거한 사실만 쓴다.
- 내부 역할 메모와 AI 운영 지시는 `docs/agent/` 에 두고 사용자 문서에 섞지 않는다.
- `development_trace` 원장, cycle report artifact, dev-console 화면은 필요한 범위에서만 사용자 언어로 설명한다.
- 로컬 프로세스 번호, 임시 URL, 특정 과거 cycle id, 실행 시점의 pass count 같은 환경 이력은 상시 문서에 남기지 않는다.

## 주제 주입 뒤 채울 내용

프로젝트 주제가 들어오면 `user-docs` 역할은 `orchestra` 가 취합한 실제 변경, 검증 결과, 사용자 반응을 기준으로 이 폴더를 갱신한다.
그때부터 문서는 사용자가 프로젝트를 이해하고 이어서 지시할 수 있게 돕는 사용자용 기록이 된다.
